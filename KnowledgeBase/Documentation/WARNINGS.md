> **Key files:** n/a (cross-cutting).

## How to use

- When you notice a problem you are **not** fixing in the current task (per the [CLAUDE.md](../../CLAUDE.md) "Analyze before editing" / "Report even when out of scope" Work Rules), add a row below — don't rely only on a turn-level mention.
- Format: `file:line (or ClassName.Method) — symptom — severity (Low/Medium/High)`.
- Remove a row only when the issue is **confirmed fixed**.
- Durable items from `TASK_PROGRESS.md`'s "Pending / out of scope" section should be mirrored here so they outlive that file.
